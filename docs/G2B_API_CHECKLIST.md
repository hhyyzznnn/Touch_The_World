# 나라장터 API 문서 확인 체크리스트

## 🔍 현재 코드에서 확인이 필요한 사항

### 1. 필드명 및 데이터 구조

#### ✅ 이미 수정 완료
- `instNm` → `dminsttNm` / `ntceInsttNm` (수요기관명/공고기관명)
- 날짜 형식: `"2025-12-09 10:00:00"` 형식 처리

#### ❓ 문서에서 확인 필요

**1.1 지역 정보 (`rgnNm`)**
- 현재: 응답에 없을 수 있음
- 확인: 언제 제공되는지, 다른 필드명인지
- 대안: `dminsttNm`에서 지역 추출 가능한지

**1.2 분류 정보 (`bidClsfcNm`)**
- 현재: 응답에 없을 수 있음
- 확인: 다른 필드명인지, 별도 API로 조회해야 하는지
- 대안: `bidMethdNm` (입찰방법)이나 다른 필드 사용 가능한지

**1.3 기초금액 (`presmptPrce`)**
- 현재: 응답에 없을 수 있음
- 확인: 별도 API (`getBidPblancListInfo[업무]BsisAmount`)로 조회해야 하는지
- 확인: 언제 제공되는지 (모든 공고에 있는지)

**1.4 상세 URL (`bidNtceDtlUrl`)**
- 현재: 응답에 없어서 생성 중
- 확인: 실제 필드명이 다른지
- 확인: 생성한 URL 형식이 올바른지

### 2. 검색 및 필터링 파라미터

#### ❓ 문서에서 확인 필요

**2.1 공고명 키워드 검색 (`bidNtceNm`)**
- 현재: API 파라미터로 전달해도 필터링 안 됨 → 클라이언트 측 필터링
- 확인: 실제로 지원하는 파라미터인지
- 확인: 다른 파라미터명인지 (예: `bidNtceNmSrch`, `keyword` 등)
- 확인: 부분 일치 vs 완전 일치

**2.2 지역 검색**
- 확인: 지역으로 필터링하는 파라미터 존재 여부
- 확인: 파라미터명 (예: `rgnNm`, `region`, `area` 등)

**2.3 분류 검색**
- 확인: 분류로 필터링하는 파라미터 존재 여부
- 확인: 파라미터명 (예: `bidClsfcNm`, `category` 등)

**2.4 예산 범위 검색**
- 확인: 최소/최대 예산으로 필터링하는 파라미터 존재 여부
- 확인: 기초금액 API와 연동 방법

### 3. 날짜 관련 파라미터

#### ❓ 문서에서 확인 필요

**3.1 날짜 형식**
- 현재: `inqryBgnDt`, `inqryEndDt`에 `yyyyMMddHHmm` 형식 사용
- 확인: 다른 형식도 지원하는지 (예: `yyyy-MM-dd`, `yyyyMMdd` 등)
- 확인: 시간 없이 날짜만 전달 가능한지

**3.2 날짜 범위 제한**
- 현재: 60일 이상 시 "입력범위값 초과 에러" 발생
- 확인: 최대 날짜 범위는 얼마인지
- 확인: 범위 제한이 있는지

**3.3 응답 날짜 형식**
- 현재: `"2025-12-09 10:00:00"` 형식
- 확인: 모든 날짜 필드가 동일한 형식인지
- 확인: 타임존 정보 포함 여부

### 4. 페이징 및 정렬

#### ❓ 문서에서 확인 필요

**4.1 정렬 순서**
- 현재: 정렬 순서 불명확
- 확인: 기본 정렬 기준 (등록일시 내림차순인지)
- 확인: 정렬 파라미터 존재 여부

**4.2 최대 조회 건수**
- 현재: `numOfRows`로 조회 중
- 확인: 최대값 제한 (예: 1000, 5000 등)
- 확인: 제한 초과 시 동작

**4.3 총 건수 (`totalCount`)**
- 확인: `totalCount`가 정확한지
- 확인: 페이징 시 총 건수 변경 가능성

### 5. 오퍼레이션별 차이점

#### ❓ 문서에서 확인 필요

**5.1 용역/공사/물품/외자 차이**
- 현재: 용역(`Servc`)만 사용 중
- 확인: 각 오퍼레이션의 응답 필드 차이
- 확인: 교육여행은 용역이 맞는지

**5.2 기초금액 조회 API**
- 확인: `getBidPblancListInfo[업무]BsisAmount` 사용법
- 확인: 공고 목록과 기초금액을 함께 조회하는 방법
- 확인: 기초금액이 없는 공고 처리 방법

**5.3 변경 이력 조회 API**
- 확인: `getBidPblancListInfoChgHstry[업무]` 사용법
- 확인: 공고 변경 시 알림 발송 여부 결정 방법

### 6. 에러 처리 및 제한사항

#### ❓ 문서에서 확인 필요

**6.1 일일 트래픽 제한**
- 화면에서 확인: 각 오퍼레이션당 1000건/일
- 확인: 제한 초과 시 에러 코드
- 확인: 제한 초과 시 대기 시간

**6.2 에러 코드**
- 현재 확인: `00`(정상), `01`(Application Error), `07`(입력범위값 초과), `08`(필수값 입력 에러), `10`(잘못된 파라미터), `20`(접근 거부)
- 확인: 다른 에러 코드 목록
- 확인: 각 에러 코드의 의미와 해결 방법

**6.3 ServiceKey 인코딩**
- 현재: URL 인코딩 없이 전달
- 확인: 인코딩 필요 여부
- 확인: 특수문자 처리 방법
